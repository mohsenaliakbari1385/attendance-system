# سیستم حضور و غیاب - دکومنتیشن امنیتی

## 🔒 بهبودی‌های امنیتی اضافه شده

### 1. سیستم ثبت‌نام معلم
- معلمان می‌توانند حساب خود را ایجاد کنند
- نیاز به ایمیل معتبر برای تأیید حساب
- رمز عبور الحق‌تر شده با bcrypt
- معتبرسازی ورودی (validation) برای نام کاربری و رمز عبور

**فایل:** `register.php`

### 2. تأیید ایمیل
- پیام تأیید خودکار به ایمیل معلم
- لینک تأیید یکتا برای هر حساب
- معلمان نمی‌توانند بدون تأیید ایمیل وارد شوند
- توکن تأیید 24 ساعت معتبر است

**فایل:** `verify-email.php`

### 3. کنترل دسترسی بر اساس نقش (RBAC)
سیستم نقش‌های زیر را اجرا می‌کند:

#### معلم (Teacher)
- ✅ ایجاد دانش‌آموز
- ✅ ایجاد کلاس
- ✅ ایجاد درس
- ✅ مشاهده تمام گزارش‌های حضور
- ✅ دانلود فایل CSV

#### دانش‌آموز (Student)
- ✅ ثبت حضور خود
- ✅ مشاهده سابقه حضور شخصی
- ❌ دیگر دانش‌آموزان را نمی‌توانند ببینند
- ❌ نمی‌توانند کلاس یا درس ایجاد کنند
- ❌ نمی‌توانند دانش‌آموز جدید ایجاد کنند

**فایل:** `api/rbac.php`

### 4. احراز هویت بهتر (Authentication)
- کلمات عبور با bcrypt رمزگذاری می‌شوند
- تأیید اعتبار سنجی قوی
- سطح دسترسی بررسی می‌شود
- رویدادهای امنیتی ثبت می‌شوند

**فایل:** `api/auth.php`

### 5. ثبت رویدادهای امنیتی
تمام اقدامات امنیتی ثبت می‌شوند:
- ورود موفق و ناموفق
- خروج
- ایجاد دانش‌آموز
- ثبت حضور
- دانلود گزارش

**فایل:** جدول `security_logs` در دیتابیس

## 🗄️ تغییرات پایگاه داده

### جدول Users (بهبودیافته)
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    fullname TEXT,
    email TEXT UNIQUE,              -- ✨ جدید
    username TEXT UNIQUE,
    password TEXT,                  -- بهتر: bcrypt
    role TEXT DEFAULT 'student',
    is_verified INTEGER DEFAULT 0,  -- ✨ جدید
    verification_token TEXT,        -- ✨ جدید
    created_at DATETIME             -- ✨ جدید
);
```

### جدول Security Logs (جدید)
```sql
CREATE TABLE security_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER,
    action TEXT,
    details TEXT,
    ip_address TEXT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 🚀 نحوه استفاده

### ثبت‌نام معلم جدید
1. بر روی "اینجا ثبت‌نام کنید" کلیک کنید
2. فرم ثبت‌نام را پر کنید
3. ایمیل خود را بررسی کنید
4. بر روی لینک تأیید کلیک کنید
5. اکنون می‌توانید وارد شوید

### ورود به سیستم
1. نام کاربری و رمز عبور خود را وارد کنید
2. بر روی "ورود به سیستم" کلیک کنید

### اطلاعات ورود پیش‌فرض (Admin)
- **نام کاربری:** admin
- **رمز عبور:** admin

## 🔐 مميزات امنیتی

### 1. رمزگذاری کلمات عبور
- استفاده از `password_hash()` و `password_verify()` (bcrypt)
- کلمات عبور قدیمی (md5) نیز پذیرفته می‌شوند (سازگاری)

### 2. معتبرسازی ورودی
- بررسی نام کاربری (3-20 کاراکتر، الفانومریک و _ فقط)
- بررسی طول رمز عبور (حداقل 6 کاراکتر)
- معتبرسازی ایمیل

### 3. کنترل دسترسی
- دانش‌آموزان تنها می‌توانند حضور خود را ببینند
- دانش‌آموزان نمی‌توانند حضور دیگران را مشاهده کنند
- معلمان فقط می‌توانند دانش‌آموزان را ایجاد کنند

### 4. جلوگیری از حملات
- اسکیپ خودکار خروجی (htmlspecialchars)
- استفاده از Prepared Statements

## 🛡️ بهترین روش‌ها

### برای توسعه‌دهندگان
1. هرگز از md5 برای کلمات عبور استفاده نکنید
2. از Prepared Statements استفاده کنید
3. ورودی‌های کاربر را همیشه معتبر‌سازی کنید
4. خروجی‌های HTML را اسکیپ کنید
5. رویدادهای امنیتی را ثبت کنید

### برای کاربران
1. رمز عبور قوی تنظیم کنید
2. ایمیل خود را نزد شما نگه دارید
3. پس از ورود، خروج کنید
4. حساب خود را به کسی نگویید
