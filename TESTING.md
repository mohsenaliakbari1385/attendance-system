# دستور تست سیستم

## ✅ تست‌های امنیتی

### 1. تست ثبت‌نام
```
[ ] بازدید از register.php
[ ] فرم ثبت‌نام را با اطلاعات معتبر پر کنید
[ ] پیام تأیید ایمیل دریافت شود
[ ] لینک تأیید کار کند
[ ] بعد از تأیید، ورود ممکن شود
```

### 2. تست احراز هویت
```
[ ] ورود با نام کاربری/رمز عبور درست
[ ] ورود با نام کاربری/رمز عبور غلط ناموفق باشد
[ ] ورود بدون تأیید ایمیل ناموفق باشد
[ ] خروج کار کند
```

### 3. تست RBAC - معلم
```
[ ] معلم بتواند دانش‌آموز ایجاد کند
[ ] معلم بتواند کلاس ایجاد کند
[ ] معلم بتواند درس ایجاد کند
[ ] معلم تمام گزارش‌های حضور را ببیند
[ ] معلم بتواند CSV دانلود کند
```

### 4. تست RBAC - دانش‌آموز
```
[ ] دانش‌آموز فقط حضور خود را ببیند
[ ] دانش‌آموز بتواند حضور ثبت کند
[ ] دانش‌آموز دکمه ایجاد دانش‌آموز نداشته باشد
[ ] دانش‌آموز دکمه ایجاد کلاس نداشته باشد
[ ] دانش‌آموز دکمه دانلود CSV نداشته باشد
```

### 5. تست جلوگیری از دسترسی غیرمجاز
```
[ ] دانش‌آموز نتواند `?export=1` را استفاده کند
[ ] دانش‌آموز نتواند درس دیگری ایجاد کند
[ ] دانش‌آموز فقط حضور خود را مشاهده کند
```

### 6. تست معتبرسازی
```
[ ] نام کاربری کوتاه رد شود (< 3 کاراکتر)
[ ] نام کاربری طولانی رد شود (> 20 کاراکتر)
[ ] نام کاربری با کاراکتر غیر معتبر رد شود
[ ] رمز عبور کوتاه رد شود (< 6 کاراکتر)
[ ] ایمیل غیر معتبر رد شود
```

## 🧪 تست منual

### سناریو 1: ثبت‌نام معلم جدید
1. `http://localhost/jasem/register.php` را باز کنید
2. اطلاعات را وارد کنید:
   - نام کامل: علی رسولی
   - ایمیل: ali@example.com
   - نام کاربری: alirasoli
   - رمز عبور: password123
3. ایمیل تأیید را بررسی کنید
4. لینک تأیید را کلیک کنید
5. وارد شوید

### سناریو 2: معلم - ایجاد دانش‌آموز
1. وارد شوید (معلم)
2. فرم "ایجاد دانش‌آموز" را پر کنید
3. دانش‌آموز اضافه شود
4. دانش‌آموز باید در جدول گزارش ظاهر شود

### سناریو 3: دانش‌آموز - ثبت حضور
1. خروج از حساب معلم
2. وارد شوید (دانش‌آموز)
3. درس را انتخاب کنید
4. "ثبت حضور" را کلیک کنید
5. حضور در جدول ظاهر شود

### سناریو 4: دانش‌آموز - نمی‌تواند CSV دانلود کند
1. وارد شوید (دانش‌آموز)
2. URL را تغییر دهید: `?export=1`
3. دسترسی رد شود

## 🔍 بررسی پایگاه داده

### دیتابیس را بررسی کنید:
```bash
sqlite3 attendance.db
```

### کاربران:
```sql
SELECT id, fullname, email, role, is_verified FROM users;
```

### رویدادهای امنیتی:
```sql
SELECT * FROM security_logs ORDER BY timestamp DESC;
```

### حضور:
```sql
SELECT u.fullname, c.course_name, a.status, a.attendance_date 
FROM attendance a 
JOIN users u ON a.student_id = u.id 
JOIN courses c ON a.course_id = c.id 
ORDER BY a.attendance_date DESC;
```
